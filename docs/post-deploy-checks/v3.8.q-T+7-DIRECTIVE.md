## Why this is a paste-ready file instead of a scheduled cron

The CronCreate runtime in this Claude Code environment returned "Session-only (not written to disk, dies when Claude exits)" even with `durable: true` passed — so an in-session schedule won't survive a VS Code restart, machine reboot, or session timeout in the 7-day window. Manual re-prompt on the target date is more reliable than a fragile in-session schedule.

If a future Claude Code release surfaces a true durable-cron mechanism, this directive can be reused there directly.
